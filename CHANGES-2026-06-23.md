# Onboarding prototype — change log

**Date:** 2026-06-23
**File:** `index-explore.html` (Project #4 – Onboarding, exploration prototype)
**Context:** Static, single-file HTML/CSS/JS prototype of Loop's business-account onboarding flow. No build system — everything lives in one file and is verified by opening/screenshotting it directly.

Changes below are grouped by design intent (not the order they were made), with a one-line rationale for each so they're case-study ready.

---

## 1. Fewer fields — lean into Canada-only

- **Removed "Country of incorporation"** field. *Loop only onboards Canadian entities today; the backend defaults to Canada, so asking is redundant friction.*
- **Primary business address now shows Canada as a fixed, non-editable default** (a locked "🇨🇦 Canada" field) instead of a selectable option. *Same reasoning — remove a choice that has only one valid answer.*
- **Removed the "Business entity ID / registration number"** field. *Not required at this stage; cutting it shortens the first page.*

## 2. Clearer form pattern

- **Moved every question/label above its input** (previously floating labels *inside* the field). *Labels-above stay visible while typing, read more clearly, and are more accessible.*
- **Sorted the Industry and Sub-industry dropdowns alphabetically** (with "Other" pinned last). *Faster to scan and find.*

## 3. Copy & voice

- **Switched the entire app to sentence case** (e.g. "Business Information" → "Business information"). Proper nouns/acronyms kept (Loop, Canada, CRA, IDV, etc.). *Calmer, more consistent voice.*
- Trimmed redundant copy:
  - Welcome subtitle: dropped "Five short pages."
  - Save note: "Your progress saves automatically — leave and finish anytime." → **"Your progress saves automatically"**
  - Primary CTA: "Save and continue →" → **"Continue →"** (incl. the incomplete-fields modal button)
  - Removed the sign-in subtitle "Pick up where you left off."

## 4. Consistent collaboration naming

- Unified every entry point for inviting a teammate to **"Add a collaborator"** (previously a mix of "Invite a collaborator" and "Manage collaborator access"). *One action should have one name.* The distinct "Invite a key contact to fill" (a person being ID-verified) was intentionally left alone.

## 5. Security step (two-step verification)

- "Skip for now" link simplified, then given a **confirmation/error state**: the first tap surfaces a red warning ("Two-step verification keeps your account secure…") and the link becomes **"Skip anyway"**; a second tap proceeds. *Discourages skipping 2FA without hard-blocking it.*

## 6. Visual & polish

- **New brand gradient** (`Gradient6.png`) on the right panel of the sign-up, sign-in, and "Add a second step" screens.
- **New step icons** on the welcome screen (Business info / Business contacts / Documents).
- **Referral code** is now progressive disclosure — hidden behind an "+ Add referral code" link on sign-up so it doesn't clutter the default form.
- **"Working on this with someone?"** moved out of a gray info box onto the open white canvas, with adjusted heading size/hierarchy.
- Fixed a **tooltip that was clipping off-screen** on the Business contacts page (wider box, more padding, plus a small JS helper that keeps it inside the viewport).
- "every" is now **bold** instead of underlined in the related-persons instructions.

## Tools & process

- **Single-file HTML/CSS/JS prototype** — all markup, styles, and interaction logic in `index-explore.html`; no framework or build step.
- **Headless Chrome screenshots** for visual QA, since there's no dev server. Each changed screen was rendered and reviewed before moving on.
- **Node "driver" scripts** — small throwaway copies of the file that jump straight to a specific screen/state (e.g. the contacts page, or the MFA skip-warning) so those states could be screenshotted without clicking through the whole flow each time.
- **Git** — the prototype is a version-controlled private repo, so each iteration is a commit rather than a duplicated folder.
- **Source of truth:** Loop's existing Storybook/codebase and the Figma onboarding files — the prototype is reconstructed to match them, so design decisions trace back there.
